# vLLM with Ray Backend — Setup Guide

## Step 1 — Activate the environment

```bash
conda activate ray-env
```

---

## Step 1.5 — Set environment variables in `~/.bashrc`

Each node type has a different physical NIC. Add the correct block to `~/.bashrc`
on each node, then `source ~/.bashrc`.

> **Why this matters:** Every node also has Docker bridge interfaces (`docker0`,
> `br-*`) in 172.x.x.x ranges. If `GLOO_SOCKET_IFNAME` is not set in the Ray
> worker process, Gloo auto-detects and picks one of those bridges — which is
> unreachable from other cluster nodes, causing `connectFullMesh` failures.

**RTX 4500 nodes — pc1…pc5 (192.168.3.71–75), interface `enp0s31f6`**
```bash
export GLOO_SOCKET_IFNAME=enp0s31f6
export NCCL_SOCKET_IFNAME=enp0s31f6
export NCCL_IB_DISABLE=1
export VLLM_HOST_IP=192.168.3.73
```

**RTX 3090 nodes — pc6…pc8 (192.168.3.76–78), interface `eno1`**
```bash
export GLOO_SOCKET_IFNAME=eno1
export NCCL_SOCKET_IFNAME=eno1
export NCCL_IB_DISABLE=1
export VLLM_HOST_IP=192.168.3.73
```

> **Critical — Ray worker restart required after any env var change.**
> The Ray worker process inherits env vars only at startup. A running worker
> does not pick up `.bashrc` changes. After editing `.bashrc` on any node,
> restart that node's Ray worker from a shell where the vars are sourced:
> ```bash
> ray stop
> ray start --address=192.168.3.73:6379
> ```
> Do this on every **worker** node (pc1–pc2, pc4–pc8). Do **not** run
> `ray stop` on the head node (pc3) — it tears down the whole cluster.

## Step 2 — Verify (both must print correctly)

```bash
echo "$GLOO_SOCKET_IFNAME $VLLM_HOST_IP"
# RTX 4500 nodes: enp0s31f6 192.168.3.73
# RTX 3090 nodes: eno1 192.168.3.73

which vllm
# expected: /home/user/anaconda3/envs/ray-env/bin/vllm
```

---

## Step 3 — Free the port + GPUs

A server may be running right now — stop it before launching a new one.

```bash
sudo systemctl stop vllm 2>/dev/null   # stop the systemd unit if active
pkill -f "vllm serve"                  # stop any manually started instance
sudo fuser -k 8000/tcp 2>/dev/null     # release port 8000 if still bound
sleep 5
ray status | grep GPU                  # should show: 0.0/6.0 GPU
```

> If it still shows `6.0/6.0 GPU` reserved, wait ~10 s and re-run
> `ray status | grep GPU` until it drops to `0.0/6.0`.

---

## Step 4 — Launch from pc3-4500 (head node)

The driver (rank 0) reads `GLOO_SOCKET_IFNAME` from its own shell. Remote
workers read it from their Ray process environment (set at `ray start` time).
Both must be set correctly — see Step 1.5 above.

```bash
RAY_ADDRESS=192.168.3.73:6379 VLLM_PLUGINS="" \
vllm serve ~/projects/vllm-deployment/vllm/models/3.1-8b-instruct \
  --dtype float16 \
  --tensor-parallel-size 1 \
  --pipeline-parallel-size 6 \
  --distributed-executor-backend ray \
  --gpu-memory-utilization 0.4 \
  --max-model-len 1024 \
  --max-num-seqs 4 \
  --enforce-eager \
  --host 0.0.0.0 \
  --port 8000
```

Server is ready when you see:
```
INFO: Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```
