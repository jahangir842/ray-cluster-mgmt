# vLLM on a Single Node

Serves a large language model on **one GPU** using vLLM's default local
executor — no Ray cluster required.

---

## Prerequisites

- Conda environment activated: `conda activate ray-env`
- Model downloaded to `~/projects/vllm-deployment/vllm/models/3.1-8b-instruct`
- Port 8000 free (see Step 3 if not)

---

## Memory budget

Llama 3.1-8B in `float16` occupies **~15 GB of VRAM**. On a 24 GB GPU
(RTX 4500 Ada / RTX 3090) the budget looks like this:

| Item | Size |
|------|------|
| Model weights (float16) | ~15.0 GB |
| Activation / overhead | ~0.3 GB |
| **KV cache (remainder)** | **~6.3 GB** |
| Total reserved at `--gpu-memory-utilization 0.9` | 21.6 GB |

`--gpu-memory-utilization 0.4` (the multi-node value) only reserves 9.6 GB —
less than the weights alone — so vLLM cannot allocate any KV cache blocks and
fails immediately.

---

## Step 1 — Launch vLLM

Run on **pc3-4500** (or any single node with sufficient VRAM):

```bash
conda activate ray-env

vllm serve ~/projects/vllm-deployment/vllm/models/3.1-8b-instruct \
  --dtype float16 \
  --gpu-memory-utilization 0.9 \
  --max-model-len 1024 \
  --enforce-eager \
  --host 0.0.0.0 \
  --port 8000
```

Server is ready when you see:

```
INFO: Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

### Flag notes

| Flag | Value | Why |
|------|-------|-----|
| `--gpu-memory-utilization` | `0.9` | Reserves 90% of VRAM; weights take ~15 GB so less than ~0.65 leaves no room for KV cache |
| `--enforce-eager` | — | Skips CUDA graph capture, reducing peak memory during startup |
| `--max-model-len` | `1024` | Caps context length; longer sequences consume more KV cache |
| `--max-num-seqs` | `4` | Limits concurrent requests; lower = less KV cache pressure |

## Step 2 — Test the server

```bash
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "/home/user/projects/vllm-deployment/vllm/models/3.1-8b-instruct",
    "messages": [{"role": "user", "content": "Hello, what is Ray?"}],
    "max_tokens": 200
  }'
```

List available models:

```bash
curl http://localhost:8000/v1/models
```

## Troubleshooting

### `ValueError: No available memory for the cache blocks`

`--gpu-memory-utilization` is too low. Check the startup log for the actual
numbers:

```
model weights take X GiB; ... the rest of the memory reserved for KV Cache is -Y GiB.
```

A negative KV cache value means the weights alone exceed the reserved budget.
Increase `--gpu-memory-utilization` (up to `0.95`) or lower `--max-model-len`.

### Port 8000 already in use

```bash
sudo fuser -k 8000/tcp
```

### CUDA out of memory during model load

The GPU does not have enough free VRAM. Check for other processes holding the
GPU before launching:

```bash
nvidia-smi
```

Kill any stale vLLM or Python processes, then retry.
