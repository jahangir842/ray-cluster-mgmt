# Serving Llama 3.1-8B with HuggingFace Transformers

FastAPI server with an OpenAI-compatible `/v1/chat/completions` endpoint,
using `transformers` instead of vLLM. The server code is in [`serve.py`](serve.py).

> For higher throughput and continuous batching, prefer
> [`2.vllm-on-single-node.md`](2.vllm-on-single-node.md).

---

## Prerequisites

```bash
conda activate ray-env
pip install transformers accelerate fastapi uvicorn
```

**VRAM:** Llama 3.1-8B in float16 requires ~15 GB — a 24 GB GPU (RTX 4500 / RTX 3090) is the minimum.

---

## Launch

```bash
python serve.py
```

Ready when you see:

```
Model ready.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

## Transformers vs vLLM

| Feature | Transformers | vLLM |
|---------|-------------|------|
| Throughput | Low — one request at a time | High — continuous batching |
| Latency | Higher | Lower |
| Streaming | Manual (`TextIteratorStreamer`) | Built-in |
| Multi-GPU | `device_map="auto"` | `--tensor-parallel-size N` |
| Multi-node | Not supported | `--pipeline-parallel-size N` with Ray |
| Setup | Simple | Simple |

## Troubleshooting

**OOM during model load** — another process holds the GPU. Run `nvidia-smi`, kill stale processes, retry.

**Repetitive or empty output** — add to `model.generate()`:
```python
eos_token_id=tokenizer.eos_token_id,
pad_token_id=tokenizer.eos_token_id,
```

**Slow generation** — `device_map="auto"` may have offloaded layers to CPU.
Check: `{n: p.device for n, p in model.named_parameters()}` — all should show `cuda:0`.
